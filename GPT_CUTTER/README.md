# GPT CUTTER

Скрипт ограничивает объем DOM в длинной переписке и удаляет самые старые элементы из верхней части чата. Ограничение работает не по количеству сообщений, а по суммарной высоте блоков переписки.

## Что такое `turn`

`turn` — это один блок сообщения в чате.

Примеры:

- сообщение пользователя = `1 turn`
- ответ ассистента = `1 turn`

Если в чате есть 2 вопроса и 2 ответа, это `4 turn`-элемента.

## Как работает скрипт

1. Находит контейнер прокрутки чата.
2. Находит все DOM-блоки сообщений (`turn`-элементы).
3. Измеряет суммарную высоту этих элементов вместе с вертикальными `margin`.
4. Если суммарная высота превышает лимит, удаляет самые старые элементы сверху.
5. Останавливает удаление, когда:
    - суммарная высота становится не больше лимита
    - или в DOM остается минимально допустимое количество сообщений
6. Корректирует `scrollTop`, чтобы позиция прокрутки оставалась примерно на месте.
7. Повторяет очистку:
    - при старте страницы
    - по таймеру
    - после изменений DOM в SPA

## Основные настройки

### `intervalMs`

Интервал регулярной проверки и очистки DOM в миллисекундах.

### `mutationDebounceMs`

Задержка перед очисткой после серии изменений DOM. Нужна, чтобы не запускать измерение и удаление слишком часто во время перерисовки страницы.

### `maxTurnPixels`

Максимальная суммарная высота `turn`-элементов в DOM.

### `minTurns`

Минимальное количество `turn`-элементов, которое всегда остается в DOM, даже если лимит по высоте превышен.

### `pruneOnMutation`

Включает очистку после DOM-изменений.

### `debug`

Включает вывод служебных сообщений в консоль.

## Рекомендуемая стартовая настройка

Чтобы ограничивать DOM примерно до `20000px` и всегда оставлять минимум 2 вопроса и 2 ответа:

```js
const config = {
  intervalMs: 60000,
  mutationDebounceMs: 300,
  maxTurnPixels: 20000,
  minTurns: 4,
  pruneOnMutation: true,
  debug: false
};
```

## Что будет удаляться

Удаляются только старые DOM-элементы сообщений в текущей вкладке.

Сама история чата не удаляется на стороне сервиса. После перезагрузки страницы старые сообщения могут снова появиться в DOM, и скрипт снова их подрежет.

## Измерение высоты

Скрипт измеряет высоту `turn`-элементов через DOM API. Для обычного количества сообщений это недорогая операция, если делать ее не на каждый `scroll`, а по таймеру и после debounce на DOM-изменениях.

Стартовое значение `20000px` подходит как безопасная практическая точка для первых тестов.

## Ограничения

- Скрипт опирается на текущую структуру DOM страницы.
- Если селекторы сообщений или контейнера прокрутки изменятся, селекторы в коде нужно будет обновить.
- Удаление старых сообщений из DOM может влиять на поиск по странице и ручной просмотр глубокой истории в текущей вкладке.
- Если несколько последних сообщений сами по себе очень большие, лимит по высоте может быть превышен, потому что `minTurns` имеет приоритет.

## Когда использовать

Скрипт полезен, если:

- переписка становится очень длинной
- страница начинает лагать из-за большого DOM
- сообщения имеют сильно разный размер
- нужно контролировать фактический объем DOM по высоте, а не по количеству блоков

## Краткий итог

- `turn` = одно сообщение
- ограничение работает по суммарной высоте сообщений
- стартовая настройка: `maxTurnPixels: 20000`
- минимум `4 turn` всегда остается в DOM
- скрипт удаляет верхние, самые старые сообщения из DOM и оставляет последние
